3.5 ADVANCED FEATURE #4: Geo-Fencing Security

≡ƒôì Location-Based Attendance Verification

Overview

Prevent remote attendance fraud by verifying the student's physical location. Attendance is only marked if the user is within a 100-meter radius of the designated campus location. This feature addresses a critical vulnerability that judges will immediately recognize as a real-world problem.

Technical Implementation

Location Verification Pipeline:

Frontend captures GPS coordinates using navigator.geolocation API

Coordinates sent to backend with attendance request

Backend calculates distance using Haversine formula

Compare distance against configurable radius (default: 100 meters)

Reject attendance if outside boundary with detailed error message

Log location violations for security audit trail

Haversine Formula Implementation

Calculates great-circle distance between two points on Earth:

Accounts for Earth's curvature (not simple Euclidean distance)

Accurate to within ┬▒0.5% (sufficient for 100m radius)

Computationally efficient for real-time processing

Configuration Options

User Experience

Inside Boundary: Green checkmark with "Location Verified" message

Outside Boundary: Red warning: "You must be on campus to mark attendance. Distance: 250m"

GPS Unavailable: Fallback to facial recognition only with admin notification

Admin Dashboard: Map visualization showing attendance locations with heat map overlay
